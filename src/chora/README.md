# Chora (Χώρα) - Cross-Cutting Fields

**Etymology**: From Greek χώρα (khṓra) meaning "space", "place", or "field"

**Philosophical Heritage**: In Plato's Timaeus, the Chora is the receptacle or space in which all things come to be - neither being nor non-being, but the field that allows existence.

> **Complete Biological Hierarchy**: See our [Naming Guide](/docs/naming.md) for the full architectural hierarchy and etymology of all terms.

## Why Chora?

We chose Chora for our cross-cutting infrastructure because these components:

- **Pervade Everything**: Like fields in physics, they exist throughout the organism
- **Enable Existence**: They provide the substrate on which discrete components operate
- **Have No Boundaries**: Unlike organs, they aren't bounded or discrete

## What Lives Here

Chora contains all the pervasive fields that flow through our organism:

- **aither/** - Divine flows (logging, events, errors, sensitive data)
- **stroma/** - Structural matrix (types, contracts, schemas)
- **phaneron/** - Visible manifestation (configuration)
- **eidola/** - Phantoms for testing (test doubles, mocks)

## Architectural Principles

1. **No Business Logic**: Chora contains only infrastructure
2. **No Circular Dependencies**: Chora cannot import from organa
3. **Universal Access**: All components can use any chora
4. **Pure Infrastructure**: These are the physics of our system

## The Fields Within

Each subdirectory represents a different kind of pervasive field:

- Some flow (aither)
- Some structure (stroma)
- Some manifest (phaneron)
- Some simulate (eidola)

Together, they form the complete substrate on which our organism lives.
