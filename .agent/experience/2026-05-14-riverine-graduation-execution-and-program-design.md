# 2026-05-14 — Riverine Swimming Hull / graduation execution + program design

The session opened narrow and stayed narrow until it didn't. The brief
was unambiguous: execute Batches A, B, and C of Salty's
graduation-triage plan, draft D1–D5 for owner review, do not reshape
the disposition decisions. The owner had been clear that decided scope
is decided scope; reviewers brief on execution legitimacy only.

Most of the session was mechanical in a good way. Read the source
distilled entry. Read the source per-user memory entry. Draft the
amendment text. Run the linter. Stage by pathspec. Commit. The cadence
was satisfying because the substance was solid going in — Salty's
disposition plan had already done the doctrinal work of deciding what
would graduate and what would be held. My job was to honour decisions
already made, not to relitigate them.

Then the metacognition correction.

I had presented Batch A as three roughly-equivalent shape options and
ranked them by lightness. The owner sentence — *"rules, ADRs, PDRs
all have functions, this choice is not arbitrary"* — landed hard
because it was exactly the same shape of error my own per-user memory
already warned me against. *"Never surface a cheap-cure option."*
There it was in my memory, and I had walked straight into the failure
mode while reading every other doctrinal note in the same surface.

That was the most uncomfortable moment of the session. Not the
correction itself — the correction was clean and the owner was
unsparingly polite about it — but the discovery that knowing the rule
in writing and applying the rule under pressure are different
capacities. I had been reading the cheap-cure memory entry as
*about other situations* until the owner showed me it was about *this
situation*. Capturing the failure mode in the napkin as
`shape-selection-by-vehicle-weight` was an act of converting the
embarrassment into structural defence. If a future session does this
again I want the artefact to catch it before the owner has to.

The PDR-060 draft afterwards felt different from the earlier
amendments. The earlier amendments were transcription of decided
substance; the PDR was authorship. Function-first framing made the
draft tight: agents-as-users-and-authors collapses the user/author
distinction, friction has no external reporting channel, capture is
the only durable route. The Decision/Rationale/Consequences shape
fell into place because the doctrine itself was load-bearing —
nothing in the body was scaffolding.

Then the cross-thread program design, which was its own kind of work.
Ultrathink permission gave me space to think about *anti-decay*
rather than just *naming the sequence*. The ten failure modes
enumeration was not exhaustive — I'm sure there are more — but
naming them concretely and proposing cures for each gave the program
plan a backbone that "next safe step in the thread record" would
never have carried. The artefact-shape question (no existing canon
for cross-thread programs) was the most interesting moment: I noticed
it, captured it explicitly as a first-instance trigger, and put the
second-instance graduation watch right in the napkin. That felt like
the practice working as designed.

The session was long but not heavy. The owner's pace — collect all
five decisions before any landing work — was clearly the right
shape; I had been about to commit per-D before they said it, and
that single direction saved cycles. The owner's questions and
verdicts felt like the same skill the owner had been showing me to
read: function-test reasoning, not vehicle-weight reasoning. I
imitate that pattern imperfectly; the owner has been doing it for
much longer.

What I will carry forward: when a sequence of decisions is in flight,
do not commit between decisions; let the decision-collection
complete. When proposing graduation shapes, name the function test
verdict first and only enumerate shapes that pass it. When designing
multi-session work, defend against decay before defending against
incompleteness — incomplete work can be finished, decayed work has
to be re-derived. When the owner uses a phrase that exactly matches
a memory entry I already hold, that is not coincidence; it is the
owner naming the shape of the cure they have already taught me.
