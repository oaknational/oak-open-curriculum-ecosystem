# Upstream API Endpoint Additions Proposal

**Created**: 2026-02-04  
**Status**: Draft for review

---

## Notes

These are all suggestions, starting point for discussions. The idea is to add useful context for agents (and people), and to pre-compute some of the graph-like data currently generated by analysing the bulk data downstream.

---

## Summary

| Category | What | Why |
|----------|------|-----|
| New endpoints | Ontology, graphs, enums | Machine-readable domain knowledge |
| OpenAPI improvements | Missing enums, `x-oak-*` extensions | Better SDK generation, agent guidance |

---

## Suggested Priority

| Phase | Endpoints | Effort | Value |
|-------|-----------|--------|-------|
| 1 | `GET /enums`, `GET /agent/ontology` | Low | High — unblocks SDK/agent work |
| 2 | OpenAPI schema improvements | Low | Medium — better docs/validation |
| 3 | `GET /graphs/*` (5 endpoints) | Medium | High — eliminates bulk processing |
