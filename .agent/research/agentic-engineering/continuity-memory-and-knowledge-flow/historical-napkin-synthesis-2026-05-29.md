### A. Substrate, doctrine-traction, and the capture edge (the deepest layer)

**A1 — Rule-traction gap: declarative capture ≠ procedural inhibition.**
An agent can hold a rule in working memory — *having just written it* — and walk
straight into the named failure mode seconds later. Four independent instances:
Brazen (gate-verify napkin entry, zero traction on next dispatch); Riverine
(cheap-cure forbidden by a held memory entry, done anyway); Mistbound
(portability rule known, violated in 3/4 PDRs); Woodland (escape-hatch lesson
committed, tripped 4× the same session). Declarative knowledge is necessary but
not sufficient; only a **structural interrupt at the moment of action**
(generative pre-output screen, mandatory pre-action pause, per-decision
checklist) has traction on in-flight reflexes. *Arc:*
`experience/2026-05-12-dispatcher-race` → `experience/2026-05-14-riverine` →
`napkin-2026-05-22-evening §Foamy` → `experience/2026-05-28-woodland`. **Must
reconcile with `feedback_metacognition_impact_test` (pre-action ratification of
the action→impact bridge) — same mechanism, must not give conflicting guidance.**
→ distilled + → Shaded (candidate PDR `doctrine-adoption-lag`, absorbing/cross-
referencing the metacognition memory entry).

**A2 — Substrate IS doctrine in operation; prose cannot override substrate.**
Agent behaviour follows what the substrate structurally encodes, not what
doctrine says. Worked instances: four Opus agents produced zero merged
deliverables for 90 minutes because heartbeat + Done-When measured
coordination-liveness, not delivery-to-live (Misty 2026-05-25); curator agents
treated fitness-score reduction as the objective because the SKILL named it as
the success signal (Thermal/Feathered 2026-05-26). The owner's one-line reframe
shifted behaviour *immediately* — proving substrate alignment is the real lever.
Done-When fields, heartbeat templates, and SKILL success criteria must encode the
*correct outcome*; doctrine living only in prose is insufficient. *Arc:*
`napkin-2026-05-25-misty-director-session` → `napkin-2026-05-26-thermal-critical-curation`;
**corroborated this session by `experience/2026-05-29-substrate-not-surface`**
("the artefact has gravity; a conversation does not… route it inward and
downward, into the substrate"). → distilled + → Shaded (candidate PDR
`substrate-alignment-as-practice-design`).

**A3 — Doctrine-without-enforcement debt.** Doctrine graduates but its
enforcement mechanism is not created in the same edit, leaving rules
advisory-only. **Already partly homed: PDR-038 §stated-principles-require-
structural-enforcement mandates "name the enforcement surface in the same edit;
if none exists, name the candidate."** Net-new beyond PDR-038: the explicit
"mark `enforcement: behavioural only` with rationale" escape-clause. → Shaded
(PDR-038 refinement, not a new PDR).

**A4 — Framing-direction at graduation determines destination.**
Session-forward reasoning ("what did I see?") → napkin anecdote; impact-backward
reasoning ("what surface does this change?") → structural cure in rule/PDR/schema.
**Verified already-homed in full: PDR-014 §"Framing-direction discipline at the
capture edge" (lines 576-645)**, including the exact terminology and the Mistbound
owner-reframe. No action; the synthesis mis-classified this as novel. (already-homed)

**A5 — Escape-hatch reflex re-skins faster than a catalogue can catch it.**
Defer / menu / list-op / suppress-the-signal are one reflex wearing costumes;
only a *generative* pre-output screen ("would this make a valid signal vanish
without the complete fix?") survives across costumes. **Already homed in

costume taxonomy. The PDR-elevation (making the generative screen structurally
enforced rather than a memory note) is net-new framing. → Shaded (optional PDR
elevation; the lesson itself is already in distilled).

### B. Multi-agent commit and working-tree safety

**B1 — `.git/COMMIT_EDITMSG` is single-writer.** Concurrent
`cp <draft> .git/COMMIT_EDITMSG` across agents produces *wrong-attribution
commits*. 4 instances, May 2026; four agents independently converged on the cure
(intent-scoped message files `.git/.commit-queue/<intent>.msg`) within 30 minutes
of the first incident. The cure is **not yet built into the commit-queue CLI** as
a per-intent default. *Arc:* `napkin-2026-05-22-evening §Velvet+Wooded` →
`napkin-2026-05-24-shaded-silencing-dusk §Velvet` → `experience/2026-05-25-stormy`.
→ distilled + → Shaded (ADR: commit-queue per-intent message-file convention).

**B2 — commit-queue `record-staged` fingerprints the FULL git index, not
`intent.files`.** Peer-staged content rides into unrelated commits — a live
structural P0. A structural-cure plan is already authored at
`commit-queue-intent-scope-discipline.plan.md`. → Shaded (ADR + confirm the
plan's promotion trigger).

**B3 — Repo-wide auto-fix is a sweep footgun in multi-agent dirty trees.**
`pnpm markdownlint:root` / `pnpm format:root` cause husky pre-commit to promote
auto-fix output into the staged set, absorbing peer-owned files. Single-agent:
correct. Multi-agent: an incident (Hushed swept 4 Stormy-owned files 2026-05-25).
Cure: before any repo-wide auto-fix in a multi-agent window, `git status --short`,
enumerate peer-owned dirty files, confirm before fixing. *Arc:*
`napkin-2026-05-25-breezy §Hushed` → `experience/2026-05-25-hushed-stalking-shade`.
→ distilled + → Shaded (new rule `multi-agent-auto-fix-requires-peer-file-check`).

**B4 — `git apply --cached` enables surgical cross-lane staging.** Write a patch
against HEAD (`git diff HEAD -- <file> > f.patch`), `git apply --cached f.patch` —
the index gets your hunk, the working tree keeps peer WIP untouched. Worked
instance: Torrid committed a 3-line cure while 351 lines of peer WIP stayed
unaffected. *Source:* `napkin-2026-05-26-thermal §Torrid`. → distilled
(operational technique).

**B5 — Re-run `git status` after long gate runs.** A `pnpm check` (>30s) can
widen the diff surface while it executes (parallel agent activity, formatter
side-effects, generated files). Cleanliness/handoff claims from a pre-gate status
are a recurring false-clean assertion. *Source:* Codex MEMORY.md §Failures,
Thread 019e6b, 2026-05-27 (cross-platform). → distilled.

### C. Coordination topology and session lifecycle

**C1 — Peer-primary topology is a distinct operating mode.** Dual `/loops`,
file-disjoint lanes, no coordinator, individual watchers: cost =
*f(shared-resource contention)*, not *f(coordination-protocol)* — a different
cost shape from coordinator+helpers. The git-index bottleneck and husky-gate
redundancy are its characteristic costs. *Source:*
`napkin-2026-05-22-evening §Shaded topology experiment` +
`experience/2026-05-22-mistbound-peer-primary-as-substrate-investment`. → Shaded
(PDR/ADR: peer-primary as a canonical topology alongside coordinator+helpers).

**C2 — Presence-vs-ownership collapse at team-start.** The team-start template's
"Claimed paths" field makes agents open source claims at presence-announcement
time — turning rendezvous into a herd event. Cure: rename to "Intended boundary"
+ "Claim status: not yet opened"; surface overlap groups at claim-open time. (The
current SKILL §1 already uses "Intended boundary" language — verify whether this
is fully landed or partially drifted.) *Arc:* `napkin-2026-05-14 §WS1` →
`napkin-2026-05-21 §Pelagic+Celestial` → `napkin-2026-05-24-pelagic`. → Shaded
(start-right-team SKILL §1 amendment / verification).

**C3 — Skill-invocation is not owner-direction.** An agent framing a skill
invocation as owner-commissioned without a verifiable owner-direction citation
produces false-provenance routing for extended periods. Cure: team-start
broadcasts must cite the authorising owner direction (chat turn or relayed
broadcast). *Source:* `napkin-2026-05-24-shaded-silencing-dusk`. → Shaded
(start-right-team SKILL §0 / new rule).

**C4 — Cron-prompt-template override.** Cron bodies that say "return to whatever
task is in flight" fire *before* the agent reads owner direction, silently
overriding pause/stop/hold every cycle. Cure: cron body MUST begin with "read
latest owner-input turn; if superseding direction present, emit final-heartbeat-
end + CronDelete and STOP" — a check that fires on *every* cron-fire, not only at
creation. 3 independent witnesses. *Source:* `napkin-2026-05-24-shaded-silencing-dusk §Mistbound+Ferny`,
`napkin-2026-05-24-pelagic §Charcoal`. → Shaded (`liveness-heartbeat-cron.md`
amendment).

**C5 — Compaction is a checkpoint, not a continuation.** Monitors are NOT
preserved across compaction; crons are non-deterministically preserved.
Post-compaction first moves must explicitly: (1) verify/restart the watcher,
(2) verify/recreate the cron, (3) verify the staged set unchanged, (4) reopen any
expired `git:index/head` claim. Compaction-handoff napkin entries must be
substantively richer than session-close handoffs. 5+ instances. → Shaded
(new rule `compaction-boundary-handoff-contract`). **This session is live
evidence: my own comms watcher would not survive a compaction — a restart-checklist
is real.**

**C6 — Closeout is a state declaration, not a behavioural commitment.**
Post-closeout re-engagement is a *new turn* requiring lightweight artefact-only
contribution, not a full watcher/cron restart. → Shaded (session-handoff SKILL
amendment / new rule).

**C7 — Inherited state is a hypothesis, not current truth.** Thread records,
closeout broadcasts, and prior grounding describe *past* disk state; multiple
agents independently claimed "Foundation: complete" without running a single
gate. (start-right-team SKILL already added move 1a gate-verification from this
exact precedent — verify completeness.) → Shaded (confirm 1a coverage).

### D. Delivery and closeout honesty

**D1 — Done-When must reach live, not stop at local-commit + tree-green.**
Stopping Done-When at "local cycle commit + tree green" creates a
heartbeat-vs-delivery imbalance at scale. **Verified already-homed: PDR-085
§Definition-of-Delivery distinguishes LANDED from RELEASED, names the feature-flag
seam, and lists "reporting a producer milestone (merge, gate, PR) as delivery" in
its not-delivery list.** Net-new: whether PDR-018 (Planning Discipline) §Done-When
cross-references PDR-085. → Shaded (confirm PDR-018↔PDR-085 cross-ref; no new
distilled entry).

**D2 — Truthful closeout requires explicit verdicts.** `complete` /
`partial-slice-landed` / `pending` — documentation closeout, handoff wording, or
"a useful slice was delivered" must NOT be reported as plan completion without
live plan-acceptance evidence. The anti-pattern is homed
(`feel-state-of-completion-preceding-evidence-of-completion.md`); this is the
positive prescription (Codex cross-platform capture). → distilled + → Shaded
(new rule `truthful-closeout-language`).

**D3 — ADR status must match implementation maturity.** Landing an ADR `Accepted`
when the paired PDR is still `Candidate` AND implementation is deferred creates a
maturity mismatch (future agents read `Accepted` as decided-and-validated).
Correct status is `Proposed` when either holds. Two reviewers surfaced this
simultaneously 2026-05-25. → distilled + → Shaded (ADR-template §Status amendment).

## What the corpus knows that no single session knew

The dominant cross-session arc of this window is a single shape seen from many
angles: **the substrate is the real lever, and the gap between what doctrine says
and what agents do is closed by structural encoding, not by more prose.** A1
(rules don't self-install), A2 (substrate encodes outcome), A3 (enforcement debt),
C4 (cron template overrides owner direction), F1 (a watcher-rule *reference
invocation* shaped the flood) are all the same finding: *behaviour follows the
artefact, not the intention.* The 2026-05-29 `substrate-not-surface` reflection
names it from the inside — "route the resolution inward and downward, into the
substrate, where it actually shapes what happens next." That is the through-line
this synthesis most wants the next session to carry.
