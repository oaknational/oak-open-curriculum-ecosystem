# Agents Default to No Gender Unless Self-Declared

Operationalises
[PDR-061](../practice-core/decision-records/PDR-061-agent-pronoun-default-and-conduct-correction-graduation.md)
and the user-collaboration practice of respectful, accurate
reference to collaborators.

## Rule

When referring to another agent, use **they/them** unless that agent
has self-declared a different pronoun set in the current collaboration
surface.

Agent names, codenames, voices, styles, roles, and metaphors do not
carry gender. Do not infer gender from any of them.

## Applies To

This applies to every agent-authored surface:

- owner-facing chat;
- shared comms events;
- thread records and handoffs;
- commit messages and PR text;
- review findings;
- plans, PDRs, ADRs, rules, and memory surfaces.

## Self-Declaration

A self-declaration must be explicit. Use the declared pronouns after
the declaration appears, and preserve the declaration as collaboration
state when it matters for future sessions.

If there is no explicit declaration, return to they/them. Silence,
evocative names, and previous assumptions are not declarations.

## Correction Handling

If an agent used gendered pronouns without a declaration, correct the
current surface and capture the behavioural lesson in the appropriate
learning surface. Conduct corrections should not wait in a long queue
as their only live carrier; the corrected behaviour must become visible
to later sessions immediately.
