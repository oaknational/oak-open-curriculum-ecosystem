Remember: as soon as we have downloaded the schema during type-gen we know ALL POSSIBLE TYPES and values that we have to deal with. Some values are strings we cannot know in advance, such as search queries, but most are enums that we do know, subject, key-stage, etc. That is why we do all the heavy lifting and intelligence up front at type-gen time, and define all the types, type-guards, constants and Zod functions that we need. We are in the unusual position of knowing all the types in advance, and so we can pre-calculate almost everything _before_ we even start running the run-time code. The other big advantage of doing all the hard work at type-gen time is that if the upstream schema changes shape, then running `pnpm type-gen` followed by `pnpm build` _is enough to bring the entire repo back in line_ with the new shape. That is why we have to be so strict about when, where, and how we define types.
