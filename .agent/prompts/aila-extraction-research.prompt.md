## Strategic Intent

The Oak AI Lesson Assistant contains years of accumulated pedagogical expertise, prompt engineering refinement, and educational domain knowledge. **This is not a code migration exercise.** This is knowledge archaeology.

The goal is to:

1. **Understand** the domain knowledge encoded in their implementation
2. **Extract** the pedagogical principles and educational patterns
3. **Document** thoroughly so it can inform future work
4. **Recommend** what is worth rebuilding to our standards

Future implementation (not part of this research) will rebuild extracted knowledge to our standards: schema-first, TDD, pure functions, type preservation.

Throughout the research, continuously ask:

- **What domain knowledge does this encode?** (not just "what does this code do")
- **What pedagogical principles are embedded here?**
- **Is this essential or accidental complexity?**
- **What is worth extracting and rebuilding properly?**
