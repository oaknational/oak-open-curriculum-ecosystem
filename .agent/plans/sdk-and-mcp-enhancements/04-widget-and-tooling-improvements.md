## Analysis: What the API Actually Returns

Based on the OpenAPI schema (`api-schema-sdk.json`), here's what the search endpoints return:

### `/search/lessons` (LessonSearchResponseSchema)

```json
{
  "type": "array",
  "items": {
    "properties": {
      "lessonSlug": { "type": "string" },
      "lessonTitle": { "type": "string" },
      "similarity": { "type": "number" },
      "units": {
        "type": "array",
        "items": {
          "unitSlug": "string",
          "unitTitle": "string",
          "examBoardTitle": "string | null",
          "keyStageSlug": "string",
          "subjectSlug": "string"
        }
      },
      "canonicalUrl": { "type": "string", "description": "Generated by SDK" }
    }
  }
}
```

### `/search/transcripts` (SearchTranscriptResponseSchema)

```json
{
  "type": "array",
  "items": {
    "properties": {
      "lessonTitle": { "type": "string" },
      "lessonSlug": { "type": "string" },
      "transcriptSnippet": { "type": "string" },
      "canonicalUrl": { "type": "string", "description": "Generated by SDK" }
    }
  }
}
```

### Key Finding: Why canonicalUrl is Missing

The `augmentResponseWithCanonicalUrl` function in `response-augmentation.ts` **does not process arrays**:

```typescript
// Line 40-42 in response-augmentation.ts
if (!response || typeof response !== 'object' || Array.isArray(response)) {
  return {}; // Arrays are returned unchanged!
}
```

Since search endpoints return arrays, no lesson/transcript gets augmented with `canonicalUrl`.

### Key Finding: Units Data is Available

Each lesson search result contains a `units[]` array with:

- `keyStageSlug` - e.g., "ks3"
- `subjectSlug` - e.g., "maths"
- `unitTitle` - e.g., "Angles"

The widget renderer currently looks for `l.subjectTitle` and `l.keyStage` which don't exist at the lesson root level - they're inside `units[0]`.

// In response-augmentation.ts

/**
 * Type guard to check if response is an array of lesson-like objects
 */
function isLessonArray(response: unknown): response is Array<{ lessonSlug?: string }> {
  return (
    Array.isArray(response) &&
    response.length > 0 &&
    typeof response[0] === 'object' &&
    response[0] !== null &&
    'lessonSlug' in response[0]
  );
}

/**
 * Augments array responses (like search results) with canonicalUrl on each item
 */
function augmentArrayWithCanonicalUrls<T extends { lessonSlug?: string }>(
  response: T[],
): Array<T & { canonicalUrl?: string }> {
  return response.map((item) => {
    if (item.lessonSlug) {
      return {
        ...item,
        canonicalUrl: `https://www.thenational.academy/teachers/lessons/${item.lessonSlug}`,
      };
    }
    return item;
  });
}

// Update augmentResponseWithCanonicalUrl to handle arrays:
export function augmentResponseWithCanonicalUrl<T extends object>(
  response: T,
  path: string,
  method: HttpMethod,
): T & { canonicalUrl?: string } {
  if (method !== 'get') {
    return response;
  }

  // NEW: Handle search result arrays
  if (path.includes('/search/') && isLessonArray(response)) {
    return augmentArrayWithCanonicalUrls(response) as T & { canonicalUrl?: string };
  }

  // Existing single-object logic...
}

## Question 5 Answer: Do Search Tools Return Units?

**Answer: NO** - There is no direct unit search endpoint.

Looking at the OpenAPI schema `/paths`:

- `/search/lessons` - searches lessons by title similarity
- `/search/transcripts` - searches transcripts by text

However, each **lesson** search result contains a `units[]` array showing which unit(s) the lesson belongs to. This provides indirect unit information but is not a unit search.

If unit search is needed in the future, this would require an upstream API enhancement.
