# Knowledge Graph Adoption

This subthread owns sector-facing plans that help external organisations use
Oak's knowledge graph assets in their own projects.

It is separate from
[knowledge-graph-integration/](../../knowledge-graph-integration/), which owns
internal integration into Oak MCP/search/QA systems, and separate from
[external-knowledge-sources/](../external-knowledge-sources/), which owns
third-party data sources that Oak may consume.

## Scope

In scope:

- adoption playbooks for MATs, edtechs, researchers, and public-sector
  partners;
- packaging and explanation of Oak KG assets, ontology distributions, SDK
  surfaces, MCP resources, semantic search projections, and provenance notes;
- external organisation requirements that should feed back into Oak's KG/API
  roadmap.

Out of scope:

- implementation of Oak KG runtime surfaces;
- the monorepo/workspace decision for the ontology repo;
- importing third-party knowledge graphs into Oak applications.
