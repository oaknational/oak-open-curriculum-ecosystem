### 🔴 Architectural Issue: Circular Dependency

**File**: `packages/sdks/oak-curriculum-sdk/type-gen/typegen/mcp-tools/parts/generate-synonyms-file.ts`

**Line 5**:

```typescript
import { ontologyData } from '../../../../src/mcp/ontology-data.js';
```

**Problem**: Type-gen code MUST NOT import from SDK runtime code. This creates circular dependencies:

- Type-gen depends on runtime code
- Runtime code is generated by type-gen
- Build order becomes undefined

**Cardinal Rule Violation**: "Type-gen code must not depend on curriculum SDK runtime code."

## Pending Work

### 1. Fix Circular Dependency (Priority: HIGH — blocks 02b)

**Solution**: Move synonyms into type-gen-time code OR into the vocab-gen pipeline.

**Rationale**: Synonyms are static data. They should be:

1. Defined at build/generate time (not runtime)
2. Generated into runtime code
3. Never imported from runtime back into type-gen

**Options**:

| Option | Description | Effort | Recommendation |
| ------ | ----------- | ------ | -------------- |
| A. Move to type-gen | Define synonyms in `type-gen/data/synonyms/` | Medium | ⚠️ Original plan |
| B. Move to vocab-gen | Synonyms generated by `pnpm vocab-gen` pipeline | Medium | ✅ **RECOMMENDED** |
| C. Shared static package | Create `packages/data/synonyms/` used by both | Higher | ❌ Overkill |

**NEW RECOMMENDATION**: Option B — move to vocab-gen pipeline

With the comprehensive vocabulary mining pipeline (02b), synonyms should be:
1. **Mined** from bulk download keyword definitions (parsing "also known as", etc.)
2. **Enhanced** with existing manual synonyms
3. **Generated** by `pnpm vocab-gen` into the SDK
4. **Exported** to Elasticsearch by `pnpm es:setup`

This eliminates the circular dependency AND integrates synonym generation into the repeatable pipeline.

**Actions**:

1. Remove circular import in `generate-synonyms-file.ts`
2. Move existing synonym data to `scripts/vocab-gen/data/base-synonyms.ts`
3. Update `pnpm vocab-gen` to merge base synonyms with mined synonyms
4. Generate output to `packages/sdks/oak-curriculum-sdk/src/mcp/synonyms/generated/`
5. Verify with `pnpm vocab-gen && pnpm type-gen && pnpm build`

**Acceptance Criteria**:

- No imports from `src/` in `type-gen/`
- `pnpm vocab-gen && pnpm type-gen && pnpm build` succeeds
- Synonyms still work in ES
- Synonym source is now part of vocab-gen pipeline

## Synonym Flow: Current vs Target

### Current Flow (has circular dependency 🔴)

```text
packages/sdks/oak-curriculum-sdk/src/mcp/synonyms/*.ts
                    ↓
synonyms/index.ts → synonymsData
                    ↓
ontology-data.ts → ontologyData.synonyms
                    ↑
          🔴 CIRCULAR: type-gen imports this
                    ↓
synonym-export.ts → buildElasticsearchSynonyms()
                    ↓
apps/.../src/lib/elasticsearch/setup/ → pnpm es:setup
                    ↓
                Elasticsearch (oak-syns synonym set)
```

### Target Flow (integrated with vocab-gen ✅)

```text
reference/bulk_download_data/*.json (30 files)
                    ↓
        pnpm vocab-gen
                    ↓
    ┌───────────────┴───────────────┐
    │                               │
    ↓                               ↓
scripts/vocab-gen/            scripts/vocab-gen/
  data/base-synonyms.ts         extractors/keywords.ts
  (manually authored)           (parse definitions)
    │                               │
    └───────────┬───────────────────┘
                ↓
    scripts/vocab-gen/generators/synonyms.ts
    (merge base + mined synonyms)
                ↓
packages/sdks/oak-curriculum-sdk/src/mcp/synonyms/generated/
                ↓
ontology-data.ts → ontologyData.synonyms (imports generated)
                ↓
synonym-export.ts → buildElasticsearchSynonyms()
                ↓
apps/.../src/lib/elasticsearch/setup/ → pnpm es:setup
                ↓
                Elasticsearch (oak-syns synonym set)
```

**Key difference**: Synonyms now flow FROM vocab-gen pipeline, not INTO type-gen.
