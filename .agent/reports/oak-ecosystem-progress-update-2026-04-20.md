# Oak Open Curriculum Ecosystem — Progress Update (April 2026)

This work has already created real value, and it is now in a strong position
to unlock much more through continued investment. Oak has not only built
useful new capabilities on top of its open curriculum data; it has also built
the foundations for doing more of that work faster, more safely, and with
greater reuse inside Oak and across the wider sector.

The most visible value is in the product capability that now exists. The
work is public and open-licensed. It includes a generated software
development kit (SDK) that gives developers reliable, type-safe access to
Oak's curriculum API and bulk data. It includes a server built on the Model
Context Protocol (MCP), which means teachers and other users can reach Oak's
curriculum directly inside AI tools such as ChatGPT, Claude, and Cursor,
rather than having to leave those environments and start again elsewhere. It
also includes a hybrid semantic search service, plus developer tooling around
that service, so finding the right curriculum content becomes easier through
natural language, not just through direct API calls. Authentication is in
place, and the groundwork for richer visual experiences has already been laid.

That is valuable in immediate, practical ways. Teachers can access high-
quality curriculum material in the tools they are already using for planning.
Developers can build on Oak's curriculum data with much less custom
integration work. Oak gets reusable infrastructure instead of one-off
solutions. In effect, one investment is now serving several audiences at once,
and doing so in a way that lowers the cost of future product development.

There is a second form of value in how the work has been engineered. The
modular structure, strict boundaries, disciplined documentation, automated
checks, and adapter approach to third-party services are not just signs of
careful execution. They are part of the point. The engineering has been
organised deliberately so that pieces can be reused, recombined, and extended
without the whole system becoming fragile. That matters for Oak because it
should make future product work faster and cheaper. It also matters outside
Oak, because the same structure makes the work easier for others to adopt,
adapt, and build on.

There is a third form of value in the delivery system that has been developed
alongside the product work. Over time, the project has built up a reusable way
of working safely and effectively with AI coding agents: guardrails, review
patterns, continuity mechanisms, learning loops, and supporting documentation
that help agent-led work stay reliable over longer periods. In this
project, that support system is called the Practice. The name matters less
than the value. It is becoming a transferable, improving support system for
agentic engineering that can help other projects and teams work with agents
more effectively, rather than remaining useful only here.

The current focus is now quite clear. The immediate work is to complete the
remaining visibility and monitoring work around Sentry and OpenTelemetry, and
to build the user-facing search experience that shows the semantic search
capability at its best. Knowledge-graph integration is also progressing, with
the first alignment work already completed and the next step now being how
best to bring that capability into the wider system. Alongside all of this,
the agentic engineering support system continues to improve as new lessons are
turned into better guidance, better review, and stronger continuity.

Continued investment should unlock the next stage in a fairly direct way.
Completing the work now in flight opens the path to a public alpha release.
From there, the next round of investment can focus on production
authentication, operational hardening, stronger alerting, and a more polished
user-facing experience. Beyond that, the opportunity is broader again: making
the search capability reusable from the upstream Oak API, extending knowledge-
graph and related curriculum intelligence work, and continuing to strengthen
the agentic engineering support system so that more ambitious work can be done
with confidence.

Taken together, this means the project is already delivering value on several
levels at once. It is creating useful product capability today. It is creating
a modular engineering base that should accelerate future Oak products and make
reuse easier across the wider sector. And it is creating a transferable way of
working with AI coding agents that has value beyond this single project.
There is still important work ahead, but the foundations are strong, the
direction is clear, and continued investment looks set to unlock meaningful
further value rather than simply maintain what already exists.
